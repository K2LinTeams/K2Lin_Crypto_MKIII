# 用户手册 (User Manual)

本手册为您提供 K2Lin Crypto MKIII 各项功能的详细使用指南。

> **[PRTS 系统日志]:**
> 访问权限: **等级 2 (授权人员)**
> 系统模块: **标准作战操作**
> 状态: **在线**

## 1. 加密库 (Crypt / Vault)

加密库是进行加密和解密操作的核心界面，支持两种针对不同场景的模式。

### 对称加密 (Symmetric / AES-GCM)
**适用场景**: 为自己保存机密文件，或通过预先约定的密码与信任的人共享文件。

1.  **选择模式**: 点击 **Symmetric (AES)** 标签页。
2.  **设置密码**: 在 "Session Key" 栏输入高强度密码。
    > **[华法琳的医疗笔记]:** 请勿使用 "123456" 这种脆弱的密钥。如果密钥丢失，数据将永远迷失在虚空中，即使是 PRTS 也无法找回。
3.  **输入数据**:
    *   **文本**: 在输入框内键入或粘贴信息。
    *   **文件**: 将文件拖拽至输入区域。
4.  **加密**: 点击 **Encrypt** 按钮。
    *   加密结果将显示在输出区域。
    *   您可以将其复制为 Base64 字符串。
5.  **解密**:
    *   将 Base64 密文字符串粘贴到输入区域。
    *   确保输入了正确的密码。
    *   点击 **Decrypt**。

### 非对称加密 (Asymmetric / ECC / X25519)
**适用场景**: 向特定联系人发送机密消息，且无需预先共享密码。

1.  **前置条件**: 您必须已导入接收者的 **身份卡 (Identity Card)** (见第 2 节)。
2.  **选择模式**: 点击 **Asymmetric (ECC)** 标签页。
3.  **选择接收人**: 从下拉菜单中选择联系人。
4.  **输入消息**: 输入您要发送的内容。
5.  **加密**: 点击 **Encrypt**。
    *   系统将生成一个一次性的临时密钥，并针对接收者的公钥加密数据。
    *   只有持有对应私钥的人才能解密此消息。

## 2. 身份系统 (Identity)

身份系统用于验证联系人并安全地交换密钥。

### 生成身份
1.  进入 **Identity** 面板。
2.  点击 **Generate New Identity**。
3.  系统将生成一对新的 X25519 密钥对。
    *   **私钥 (Private Key)**: 安全存储在您的本地浏览器存储中。
    *   **公钥 (Public Key)**: 可以自由公开。

    > **[PRTS 警报]:** **绝对不要分享您的私钥。** 请将其视为连接至源石网络的神经接口——一旦暴露，将导致灾难性的后果。

4.  您的 **身份 ID** (如 `1234 5678 9012`) 是通过算法从您的公钥计算得出的。

### 导出身份卡
1.  点击 **Download Identity Card**。
2.  这将保存一张包含您公钥和 ID 的 PNG 图片。
3.  通过邮件、聊天软件等方式将此图片发送给您的朋友。

### 导入联系人
1.  接收朋友发来的身份卡 (PNG)。
2.  在 Identity 面板点击 **Import Contact**。
3.  选择该图片文件。
4.  该联系人将被验证并添加到您的 "已知干员 (Known Associates)" 列表中。

## 3. 伪装面板 (Mimic / Steganography)

伪装面板允许您将加密数据隐藏在看似无害的图片中。

### 嵌入数据
1.  **先行加密**: 在 **Vault** 中执行一次加密操作。结果将自动暂存以供嵌入。
2.  **进入伪装面板**: 切换到 **Mimic** 面板。
3.  **上传载体**: 上传一张普通图片 (PNG, JPG, WebP) 作为 "封面"。
4.  **嵌入**: 点击 **Embed Encrypted Data**。
    *   APP 将修改图片像素的最低有效位 (LSB) 以存储您的数据。
5.  **下载**: 点击 **Download Stego Image**。
    > **[可露希尔的工坊提示]:** 务必导出为 **PNG** 格式。JPG 等格式是 "有损" 的——它们会像压缩机下的源石虫一样粉碎隐藏的数据。
    >
    > *通过 WhatsApp/WeChat 传输也会清洗文件数据。请使用邮件或压缩包传输。*

### 提取数据
1.  **上传**: 将可疑图片上传至 Mimic 面板。
2.  **扫描**: 系统自动扫描隐藏的文件头。
3.  **结果**: 如果发现数据，系统将自动提取并将其发送回 **Vault** 进行解密。

## 4. 恐慌模式 (Panic Mode)

专为紧急隐私保护设计。

*   **触发**: 点击顶部的 **Panic** 按钮 (通常是感叹号图标)。
*   **表现**: 整个应用界面瞬间替换为一个虚假的 "游戏启动器" (风格类似《原神》)。
*   **恢复**:
    *   点击隐藏的触发区域 (通常是 "Privacy Policy" 链接)。
    *   输入您的 PIN 码 (如果在设置中配置过)。
    *   应用恢复至之前的状态。
